# 暑期实习备战指南

## 背景与现状分析

从你当前的技术栈来看，有几个明显的优势：

- Rust 作为主力语言，掌握了 trait、泛型、智能指针、多线程等高级特性
- 有 mini web、mini redis 等跟随教程的实践经历
- 坚持 LeetCode 刷题，具备算法基础
- 熟悉 Linux 环境、Docker 容器化和 SSH 远程开发
- 正在参加清华大学操作系统开放训练营，系统编程能力持续提升
- 在 vibe coding 模式下完成过两个全栈 Web 应用并完成部署

主要不足是**缺乏正式的工程项目经历**，这是面试时最需要补充的短板。

---

## 实习求职的优先级建议

根据你的技能背景和求职目标，建议按以下优先级投递：

1. **后端开发实习（首要目标）**：最贴合当前能力，可作为切入点积累工程经验
2. **infra / 系统开发实习**：清华 OS 训练营结束后，可作为重要补充方向投递
3. **Web3 / 区块链 / 量化方向**：作为中长期目标，需补充领域知识后再主攻

---

## 近期行动计划

### 第一阶段：补项目（当前 → 实习投递前）

教程跟做的项目在面试中说服力有限，需要有**自己主导的、有一定工程量的项目**。

**推荐方向（选一个重点做）：**

- **CLI 工具 / 系统小工具**：用 Rust 写一个有实际用途的命令行工具，比如简单的任务调度器、文件同步工具、进程监控器等。代码质量比功能多更重要，注重错误处理和文档。

- **改造 vibe coding 项目**：将之前用 AI 辅助完成的全栈项目，自己从头理解每一行代码，补充测试、完善 README、优化架构，将其变成能在简历上有说服力展示的项目。

- **OS 训练营配套项目**：清华训练营本身的实验项目（如 rCore 相关）在 infra 和系统方向面试中是很强的背书，完成度越高越好。

**衡量标准**：项目需要能讲清楚"我遇到了什么问题、做了什么设计决策、为什么这样做"，而不只是"我实现了什么功能"。

---

### 第二阶段：准备简历与 GitHub

**简历要点：**

- 技能部分突出 Rust 熟练程度，列出具体特性（trait object、Arc/Mutex、async/await 等）
- 项目部分每条用"做了什么 → 解决了什么问题 → 用了什么技术 → 有什么效果"的结构描述
- Docker 容器化部署、Linux 运维经验可以单独列出，这在后端和 infra 岗位是加分项

**GitHub 整理：**

- 为每个重要项目写清晰的 `README.md`，包含：项目简介、技术栈、如何本地运行、架构说明
- 保持提交记录的连贯性，避免一次性大量提交
- 考虑将 vibe coding 项目的核心代码整理后公开

---

### 第三阶段：算法面试准备

你已有刷题习惯，需要调整刷题策略：

- **优先刷高频题**：重点覆盖数组、链表、树（二叉树 DFS/BFS）、动态规划（背包、序列型）、图（BFS/DFS/拓扑排序）这几个核心类型
- **用 Rust 做题**：大厂面试通常接受 Rust，坚持用 Rust 刷题既练习语言又练算法
- **目标**：LeetCode 刷到 150 题以上中等题，重点是能在 30 分钟内清晰讲解思路

---

### 第四阶段：技术知识补充

**后端基础（必要）：**
- HTTP 协议基础：状态码、请求方法、常见 header、REST 设计风格
- 数据库基础：SQL 基本操作、索引原理、事务 ACID 概念
- 并发模型：线程 vs 协程、锁机制、常见并发问题（死锁、竞态）

**系统方向（结合 OS 训练营）：**
- 进程/线程/协程调度
- 内存管理：虚拟内存、页表、缺页中断
- I/O 模型：同步/异步 I/O、epoll 原理

**Web3 方向（如有余力）：**
- 了解 EVM 基础和 Solidity 语法
- 理解区块链核心概念：共识机制、智能合约、Gas
- 看一看 Solana 或 Sui 生态，这两个方向 Rust 开发需求较多

## 面试中的注意事项

1. **项目介绍要主动**：不要等面试官问，开场就能流畅介绍自己的核心项目（2 分钟版本）
2. **Rust 是差异化优势**：很多后端面试官没见过 Rust 候选人，提前准备好解释 Rust 所有权/借用检查的思路，展现对内存安全的理解
3. **诚实面对知识盲区**：没用过 Spring 不必强行说会，可以说"我主要用 Rust 做系统级开发，Java 生态了解不深，但学习新框架对我来说不是障碍"
4. **vibe coding 项目需要解释清楚**：面试官可能会追问每个技术细节，要确保自己能独立讲解，否则不要放在简历上

---

## 总结

当前阶段最重要的三件事，按优先级排列：

1. **做一个有说服力的自主项目**（这是简历的核心，花时间最值得）
2. **完成 OS 训练营核心实验**（系统方向的强力背书）
3. **稳步刷题，保持算法手感**

你现在的技术基础其实不弱，缺的主要是工程经验的积累和展示。坚持做完这几件事，暑期实习的成功率会大幅提升。
