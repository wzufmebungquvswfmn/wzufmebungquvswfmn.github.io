# 暑期实习求职指导：Rust 开发者的进阶路线

> 写给正在准备暑期实习的自己：从现有基础出发，制定合理可行的求职规划。

---

## 一、现状评估

**优势：**
- 掌握 Rust 基本语法及 trait、泛型、智能指针、多线程等高级特性
- 有 mini web、mini redis 的项目实践经验
- 坚持刷 LeetCode，具备一定的数据结构与算法基础

**待补强：**
- 尚无独立完成的个人项目
- 异步编程（async/await、tokio）仍在学习中
- 缺乏完整的工程项目经验（CI/CD、测试、文档、代码审查等）

---

## 二、目标优先级与可行性分析

根据你的描述，目标优先级为：

| 优先级 | 方向 | 当前可行性 |
|--------|------|-----------|
| ⭐⭐⭐ | Web3 / 区块链 / 金融 / 量化 | 中等，需要补充领域知识 |
| ⭐⭐ | Rust Infra / 系统级开发 | 中等，需要更多底层项目经验 |
| ⭐ | 大厂 Rust 后端 / 其他后端 | **最高**，当前最可行的切入点 |

**短期策略（暑期实习）：** 优先投递大厂后端岗，打好工程基础；中长期向 Web3 / 量化方向发展。这是最务实、风险最低的路径。

---

## 三、暑期实习前的准备清单（4–8 周）

### 3.1 完成一个有质量的个人项目（最重要）

这是简历上最有分量的内容。建议在现有 mini 项目基础上升级，或从零开始做一个完整项目。

**推荐方向（难度由低到高）：**

1. **升级 mini-redis**：加入持久化（AOF/RDB 简化版）、发布订阅、LRU 淘汰策略，对标真实 Redis 子集功能。写清楚 README、测试、benchmark。
2. **HTTP 代理 / 反向代理**：用 tokio + hyper 实现一个简单的反向代理，支持负载均衡（轮询/随机），加深对异步网络编程的理解。
3. **命令行工具**：用 Rust 实现一个实用的 CLI 工具（如文件搜索、日志分析、CSV 处理），展示工程规范性（clap、error handling、测试）。

**项目质量标准：**
- `README.md`：项目介绍、架构图（可用 mermaid）、运行方式、设计思路
- 完整的单元测试与集成测试
- 使用 `cargo clippy` 和 `cargo fmt` 保持代码质量
- 发布到 GitHub，打上版本 tag

### 3.2 夯实异步编程

异步是现代 Rust 后端开发的核心，面试中几乎必考。

**学习路径：**
- 理解 `async/await` 的语法糖本质（Future 状态机）
- 掌握 `tokio` 基础：`tokio::spawn`、`select!`、`mpsc channel`
- 了解 `Pin` 和自引用结构体的概念（能说清楚原理即可）
- 实践：将 mini-web 或 mini-redis 用 tokio 重写异步版本

**推荐资源：**
- [Tokio 官方教程](https://tokio.rs/tokio/tutorial)
- [Async Book（异步编程手册）](https://rust-lang.github.io/async-book/)

### 3.3 强化算法，准备笔试

大厂笔试和面试基本都有算法题。

**重点题目类型：**
- 二叉树（DFS/BFS）、图（最短路、拓扑排序）
- 动态规划（背包、区间DP、状态压缩）
- 滑动窗口、双指针
- 堆、排序、二分查找

**目标：** LeetCode 达到 200+ 题，中等题为主，确保热门 TOP 100 题目熟练。

### 3.4 了解后端工程基础

简历上写后端，需要知道这些概念：

- **HTTP 协议**：常见状态码、RESTful 设计规范、gRPC 基础
- **数据库**：SQL 基础（JOIN、索引、事务 ACID）、Redis 基本数据结构与使用场景
- **系统设计基础**：缓存策略、消息队列的使用场景、分布式 ID 生成（能说清楚思路）
- **Linux / Shell**：基本命令（grep、awk、top、netstat），了解进程、线程、文件描述符

## 四、简历优化建议

### 4.1 项目描述写法

避免写"跟随教程做了 xxx 项目"，改为突出**你做了什么决策**、**解决了什么问题**：

❌ 差的写法：
> 跟随教程实现了 mini-redis，学习了 Rust 异步编程

✅ 好的写法：
> 使用 Rust + Tokio 实现了 mini-redis 子集，支持 GET/SET/EXPIRE 命令及发布订阅功能；采用 AOF 格式实现持久化；编写 50+ 单元测试，通过 criterion 基准测试达到 X 万 QPS

**公式：** 用了什么技术 + 实现了什么功能 + 解决了什么问题 + 达到了什么效果（量化）

### 4.2 技能栏描述

- **编程语言**：Rust（熟练）、可酌情加入其他你了解的语言（Go、Python 等）
- **框架 & 工具**：Tokio、Axum/Actix-web、Git、Docker（如果会的话）
- **数据库**：MySQL / Redis（基础）
- **其他**：了解 Linux 基本操作，了解 HTTP 协议

## 六、面向 Web3 / 量化的中长期准备

虽然暑期实习主攻大厂后端，但可以在业余时间为后续方向做积累：

### Web3 / 区块链方向
- 学习 Solana 或 Substrate（都是 Rust 生态）
- 阅读 [Solana 开发文档](https://docs.solana.com)，尝试写简单的链上程序（Program）
- 了解以太坊 EVM 基础概念（即使不直接用 Rust 也需要知道）
- 关注 GitHub 上的区块链 Rust 项目（如 `near-sdk-rs`、`solana-program-library`）

### 量化 / 金融方向
- 学习统计学基础和金融基本概念（期权、期货、套利）
- Python 量化库（pandas、backtrader）与 Rust 的结合（通过 FFI 或 PyO3）
- 关注国内量化私募（幻方、九坤等）的实习机会，通常需要 C++/Rust + 算法能力

## 七、总结

```
短期目标（暑期）：
  拿下大厂后端实习 → 积累工程经验 → 完善简历

中期目标（校招）：
  向 Rust infra / 系统开发 转型 → 目标 PingCAP / 字节基础架构

长期目标：
  进入 Web3 / 量化领域 → 需要领域知识积累 + 更强的工程背景
```

**最关键的一件事：尽快做出一个完整的、有质量的个人项目。** 这是当前简历上最需要补充的短板，也是打开面试机会的钥匙。
