# 所有权
*注：这里不会系统阐释所有权的细节，只是记录一些零散的知识点*

## Clone
对于没有实现 Copy 的类型，若想达到 Copy 的效果，那就使用 Clone，但是对于实现了的类型 Copy 等价于 Clone

**Clone传入的参数是不可变借用：**
```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1.clone(); // 等价于 let s2 = s1.clone(&s1); 这显然是更清晰的写法
    println!("s1: {}, s2: {}", s1, s2);
}
```  

## 引用的互斥
一个数据只能有一个可变引用，或者多个不可变引用，即一个可变引用与其它引用互斥, 在可变引用存在时不允许其它引用存在。  

对所有者自己而言，情况则很有意思，当一个数据被声明为 mut 时，在**存在其它引用期间**，你可以根据行为来将所有者视为以下三种状态：
- 无视所有者：
当所有者在这期间什么行为都没有，所有者不被视作引用
- 不可变引用：
当所有者在这期间只读取数据，它会被当做不可变引用
- 可变引用：
当所有者在这期间修改数据，它会被当做可变引用，此时其它引用实际上是不允许存在的，否则会报错


## mut 标识
一个数据有一个所有者，mut 标识的作用是让我们能够通过所有者修改这块数据，不使用 mut 只是不能通过这个所有者修改数据，但不代表数据本身不可变，事实上既然数据在那，那么如果愿意的话，总有办法修改它的，看以下例子：

```rust
let v = vec![1, 2, 3];
for mut x in v {
    x += 1;
    print!("{} ", x);
}
```
输出为：
```
2 3 4 
```

v 被声明为不可变，但循环中仍能修改并打印，因为 for-in 结构实际上对容器进行迭代，而不加 & 相当于调用 v.into_iter()，它让 v 失去对这块内存的所有权而生成一个迭代器，迭代器里的元素的所有权在迭代过程中被转移给 x，x 又被声明为 mut，可以对每个元素进行修改。
