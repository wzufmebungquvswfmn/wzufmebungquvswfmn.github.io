# 集合数据结构
Rust 中最常用的集合类型是 `Vec` 和 `HashMap`。前者用于存储一组有顺序的数据，后者用于通过键快速查找值。很多时候，光靠这两个类型就已经能解决大部分数据存储和处理问题。

不过在刷题时，还经常会遇到队列、堆、有序映射、集合判重等场景，因此这一章主要整理 Rust 中常用集合数据结构的基本用法。这里不会完整复刻标准库文档，而是重点记录平时写题时容易查、容易忘、容易踩坑的操作。

需要注意的是，`Vec`、切片、数组、`String` 并不都在 `std::collections` 模块中，但它们同样是 Rust 中最重要的数据结构。本章暂时重点放在这些常用集合：

- [Vec](集合数据结构/vec.md)  
- [HashMap/HashSet](集合数据结构/hash.md)  
- [BinaryHeap](集合数据结构/binaryheap.md)  
- [VecDeque](集合数据结构/vecdeque.md)  
- [BTreeMap/BTreeSet](集合数据结构/btreemap.md)  

## 如何选择集合

如果只是存储一组元素，并且需要按照下标访问，优先考虑 `Vec`。

如果需要通过某个 key 快速找到对应的 value，使用 `HashMap`。

如果只关心某个元素是否出现过，使用 `HashSet`。

如果需要队列，尤其是 BFS 中从队头取元素、从队尾加入元素，使用 `VecDeque`。

如果需要每次取出最大值或最小值，使用 `BinaryHeap`。

如果需要 key 保持有序，或者需要范围查询，使用 `BTreeMap` / `BTreeSet`。

至于 `LinkedList`，Rust 标准库中确实有这个类型，但日常代码和刷题中通常很少直接使用。LeetCode 中的链表题一般是题目自定义的 `Option<Box<ListNode>>`，它的重点是所有权、`Option::take` 和节点重连，而不是 `std::collections::LinkedList`。
