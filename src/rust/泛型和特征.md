#  泛型(Generic) 和特征 (Trait)

记录一些零碎的泛型和特征的知识点

## 泛 型
泛型是一种类型参数，可以是任何类型，在编译时会被替换为具体的类型
-  `fn func<T>(arg: T) -> N;`
-  `struct S<T> { arg: T, .. };`
-  `enum E<T> { A(T), B, .. };`
-  `impl<T: 约束条件> S<T> { fn func<T>(arg: T) -> N { ..T受约束的调用.. } }`
    -  `约束条件` 为 `特征` 或 `类型` 或 `特征+类型` 或 `特征+特征` 等
    - 只有< >里面和 where 子句等处的T才能跟约束条件，函数列表中的类型参数一般跟普通类型、泛型或具体实现，如 `i32`, `T`, `impl Trait` 等

## 特征
-  `trait TraitName { fn func(&self) -> N; }`
-  `impl TraitName for S { fn func(&self) -> N { .. } }`
-  `fn func(arg: impl TraitName) <==> fn func<T: TraitName>(arg: T)`
-  `fn func(arg: impl TraitName + TraitName2) <==> fn func<T: TraitName + TraitName2>(arg: T)` //多重约束

## `Move`、`Copy` 和 `Clone`
### 简介
- `Move` 是默认行为，在栈上进行位拷贝（实际可能不进行拷贝），所有权转移，原变量失效。
- `Copy` 也在栈上进行位拷贝，原变量仍然有效。适用于完全在栈上的类型，包括固定大小的数组。它与 `Drop` 不兼容。
- `Clone` 为无法实现 `Copy` 的类型提供深拷贝，若不采用引用计数会比较昂贵。
### 派生
- 以下是常用的派生方式，适用于简单类型组合：  
    `#[derive(Copy, Clone)]`   
- 这样写的原因在于 `Copy` 继承自 `Clone`（`Copy: Clone`），所以必须同时实现 `Clone`。

## `Sized` 、`!Sized` 和 `?Sized`
- `Sized` 标记编译器已知大小固定的类型，默认所有类型都满足 Sized 约束。  
- `!Sized` 表示类型大小在编译时未知，如动态大小类型 `str` 和 `dyn Trait`。  
- `?Sized` 表示类型可能是 Sized 也可能是 !Sized，所以需要作为动态大小类型 (DST) 处理，即传参时传引用，因为地址长度总是固定的 8 字节或 16 字节，例如：
```rust
fn print_it<T: ?Sized>(arg: &T) {
    // ...
}
```

## `Send` 和 `Sync`  
这两个特征标记类型是否可以安全的在线程之间传递共享，是典型的标记特征，无具体实现。[多线程并发](多线程.md) 中有提到。  
### `Send` 
它标记一个类型可以在线程间安全的转移所有权，即可以把它 move 到其它线程里去。  

大多数 Rust 类型都是 Send 的；而典型的非 Send 类型是 `Rc<T>`，因为它的引用计数器不是线程安全的，还有裸指针 `*const T` 和 `*mut T`，因为它们没有任何所有权和生命周期的保证。
### `Sync`
标记一个类型可以在多个线程间安全的共享引用，即 `T: Sync` <=> `&T: Send`。  
典型非 Sync 的类型是 `Cell<T>` 和 `RefCell<T>`，因为它们提供了内部可变性，但没有线程安全的保证。

### 线程安全组合： 
`T: Send` => `Mutex<T>: Sync` 和 `RwLock<T>: Sync`
`T: Send + Sync` => `Arc<T>: Send + Sync`

### 实现或派生
自动派生，但如果有裸指针或 `Rc<T>`，在确保线程安全的情况下，可以手动在unsafe块中实现

## `std::io` 的核心： `Read` 和 `Write` 以及 `BufRead` 和 `Seek`
### `Read`
实现了 Read 的读者，有必需（实现的）方法 read()：
```rust
// 从 self 读取数据到 buf 中，尽可能读满并返回实际读取的字节数
fn read(&mut self, buf: &mut [u8]) -> Result<usize>;
```  
除此之外还有以下常用默认方法，它们都会尽可能读至self的EOF，并返回实际读取的字节数：
```rust
// 从 self 读取数据直到 EOF，追加到 buf 中
read_to_end(&mut self, buf: &mut Vec<u8>) -> Result<usize>  
// 从 self 读取数据直到 EOF，追加到 buf 中，要求字符是 UTF-8 编码
read_to_string(&mut self, buf: &mut String) -> Result<usize>
```

### `Write`
实现了 Write 的写者，有必需方法 write() 和 flush()：
```rust
// 从 buf 中写入数据到 self 中，返回实际写入的字节数
fn write(&mut self, buf: &[u8]) -> Result<usize>;
// 把内部缓冲区（self的一个成员）的数据强制写入底层设备，确保数据被发送或保存
fn flush(&mut self) -> Result<()>;
```

### `Seek`
它对读者或写者进行定位，有必需方法 seek()，配合枚举 `SeekFrom` 使用：
```rust
enum SeekFrom {
    Start(u64), // 从起始位置偏移
    End(i64),   // 从末尾位置偏移
    Current(i64) // 从当前位置偏移
}
// 将 self 的当前位置设置为 pos 所指定的位置，并返回新的位置
fn seek(&mut self, pos: SeekFrom) -> Result<u64>;
```

### `BufRead`
带缓冲区的读者，继承自 `Read`，有必需方法 fill_buf() 和 consume()：
```rust
fn fill_buf(&mut self) -> Result<&[u8]>; // 返回内部缓冲区中可用的数据
fn consume(&mut self, amount: usize); // 从内部缓冲区中消费 amount 字节的数据
```
常有默认方法有：
```rust
// 读取一行 UTF-8 字符到 buf 中，返回实际读取的字节数
fn read_line(&mut self, buf: &mut String) -> Result<usize>;
// 返回一个迭代器，逐行读取数据
fn lines(self) -> Lines<Self> where Self: Sized;
```  

读者需要转换为带缓冲区的读者才能使用 `BufRead` 的方法，可以通过：  
```rust
BufReader::new(inner: R) -> BufReader<R>
BufReader::with_capacity(capacity: usize, inner: R) -> BufReader<R>```
```   
来实现转换，其中 `R: Read`，`capacity` 是缓冲区的大小，默认为 8KB。  

**特别地**，对 **stdin** 来说，`lock()` 方法可以将其转换为 `BufReader`,详见[标准输入](字符串/stdin.md)

## `Default`
标记特征，便于初始化，只有一个方法：
```rust
fn default() -> Self;
```
可以直接派生 `#derive(Default)`。

## `Debug` 和 `Display`
- `Debug` 两种打印语法糖的具体实现：  
`{:?} // 格式化打印`   
`{:#?} // 漂亮打印`  
可以通过 `#[derive(Debug)]` 派生。

- `Display` 用于内容显示，使得类型可以使用 `{}` 进行格式化输出。  
需手动实现：
```rust
fn fmt(&self, f: &mut Formatter<'_>) -> std::fmt::Result {
    // 这里实现格式化逻辑，例如：
    write!(f, "MyStruct {{ field1: {}, field2: {} }}", self.field1, self.field2)
}
```
除此之外它还还赠予了 **ToString** 的实现，使得类型可以使用 `to_string()` 和 `format!()`等。

## `PartialEq` 和 `Eq`
- `PartialEq` 用于比较两个值是否相等，提供了 `==` 和 `!=` 操作符的实现，之所以叫 Partial，是因为浮点数中NaN不等于任何值，包括它自己。
- `Eq` 继承自 `PartialEq`，表示类型的值可以完全比较相等，浮点数未实现。
可以通过 `#[derive(PartialEq, Eq)]` 派生。

## `PartialOrd` 和 `Ord`
### `PartialOrd` 
- 必需方法:
```rust
fn partial_cmp(&self, other: &Rhs) -> Option<Ordering>; // 实现和任何类型右值Rhs的比较
```
附赠了 lt（<）、le（<=）、gt（>）、ge（>=）四个方法的实现  

- Ordering 是一个枚举，表示比较结果：
```rust
enum Ordering {
    Less,    // 小于
    Equal,   // 等于
    Greater, // 大于
}
```

### `Ord： Eq + PartialOrd` 
- 和 Eq 类似，必需方法:
```rust
fn cmp(&self, other: &Rhs) -> Ordering;
```
提供max、min 以及 clamp等方法的实现:
```rust
// 将 self 限制在 min 和 max 之间
fn clamp(self, min: Self, max: Self) -> Self 
where Self: Sized + PartialOrd; 
```
- `#[derive(PartialOrd, Ord)]` 可以派生, 比较按照字段在结构体中的顺序进行。

### 关联宏
```rust
#[derive(Derivative)] // 需要在 Cargo.toml 中添加 derivative 依赖
#[derivative(PartialOrd, Ord)] // 写在结构体前
struct MyStruct {
    field1: i32,
    #[derivative(PartialOrd = "ignore", Ord = "ignore")] 
    field2: String, // 忽略该字段的比较
}
``` 

## `From` 和 `Into`
- `from` 定义了如何从其它类型创建一个自己类型的值  

- `into` 恰恰相反，它定义了如何将自己类型转换为其它类型的值  

只需实现 `From` 就会自动获得 `Into` 的实现

## `AsRef` 和 `AsMut`
不同于 `From` 和 `Into`, 它们是不消耗所有权的转换:
```rust
fn as_ref(&self) -> &T;
fn as_mut(&mut self) -> &mut T;
```
这两个特征使得借用多态实现非常方便，如在函数入口加一个`.as_ref()`，就可以接受任何实现了 `AsRef` 的类型:
```rust
fn print_it(arg: impl AsRef<str>) {
    println!("{}", arg.as_ref());
}
```

## `Deref` 和 `DerefMut`
`Deref` 和 `DerefMut` 是对智能指针解引用操作符 `*` 的重载，允许我们定义自己的智能指针类型，并使其行为类似于普通引用。
### `Deref` 定义了如何通过 `*` 操作符获取智能指针内部的值的引用：
```rust
struct MyBox<T>(T);
impl<T> std::ops::Deref for MyBox<T> {
    type Target = T;
    fn deref(&self) -> &T {
        &self.0
    }
}
```
### `DerefMut` 则是 `Deref` 的mut版本，继承自 `Deref`
除了显式使用 `*` 时会解引用，在**函数传参**与**方法调用**时也会自动解引用（解引用强制转换）。

## `Hash`
### std::hash 模块下的特征，实现了 Hash 类型可以变成一个哈希值，通过下方的必须方法：
```rust
fn hash<H: Hasher>(&self, state: &mut H); // 实现了 Hasher 的 H 是一个哈希生成器
```
### 实现通过 `#[derive(Hash)]` 可以派生，或者把需要计算的字段手动哈希：
```rust
fn hash<H: Hasher>(&self, state: &mut H) {
    self.field1.hash(state); // 只哈希 field1 字段
}
``` 
- 实现了 `Hash + Eq` 的类型可以作为 HashMap、HashSet 的键。

## `Borrow`
### `borrow` 模块有下面几个 trait，主要用来扩展借用和所有权的功能
- `Borrow` 使得一个类型的引用能转换成另一个类型的引用，常见于 Hash 的键查找：
```rust
impl<Q: ?Sized> Borrow<Q> for K where K: Hash + Eq, Q: Hash + Eq {
    fn borrow(&self) -> &Q;
}
```
一个典型的例子是 `String` 实现了 `Borrow<str>`，使得对于 `get(s: &str)` 来说，传入的参数类型是 `&str` 和 `&String` 得到的哈希值相同。
- `BorrowMut` 是 `Borrow` 的 mut 版本。
- `ToOwned` 是 `Clone` 的泛用版本，提供了 `to_owned()` 方法，可以从一个引用类型直接创建一个拥有所有权的类型，例如 `str` 的 `to_owned()` 会返回一个 `String`。
- `Cow<'a,T>`（Clone on Write, 写时克隆）是一个智能指针, 在需要写时（比如调用 to_mut()）就会用 `to_owned()` 克隆出一个拥有所有权的类型，否则保持对原始数据的引用。

##  `IntoIterator` 和 `Iterator`
- `IntoIterator` 定义了如何将一个类型转换为一个迭代器，需实现 `into_iter()` 方法来返回一个迭代器，附赠了 `for` 循环的语法糖支持：
```rust
for item in collection {
    // 这里的 collection 会被自动调用 into_iter() 转换为迭代器
}
```
- `Iterator`：得到迭代器后，迭代器本身的行为就是由 `Iterator` trait 定义的了。为一个类型T实现Iterator,必须实现 `next()` 方法来返回迭代器中的下一个元素，返回值是Option<T>

想要了解更多迭代器相关的知识，可以参考[迭代器](迭代器.md)一章。
