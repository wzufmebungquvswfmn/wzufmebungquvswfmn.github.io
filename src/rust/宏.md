# 宏
本节主要介绍rust中的宏，它实现了代码->代码的转换，可以用来简化代码，提高代码的可读性。

## 宏的分类
rust中的宏分为两种，一种是声明式宏，另一种是过程宏。

- **声明式宏 (Declarative Macros)**: 使用 `macro_rules!` 定义的宏，通过模式匹配来生成代码。
- **过程宏 (Procedural Macros)**: 操作AST的宏，更强大但更复杂，需要单独的crate。

## 声明式宏

### 基本语法
宏定义的格式如下：
```rust
macro_rules! macro_name {
    // pattern => expansion
    ($arg:expr) => {
        // 宏展开后的代码
    };
}
```

### 简单示例
```rust
// 定义一个简单的宏，用于计算平方
macro_rules! square {
    ($x:expr) => {
        $x * $x
    };
}

fn main() {
    let num = 5;
    println!("Square of {} is {}", num, square!(num));
}
```

### 元变量类型
在宏中，可以使用不同的元变量类型：

| 元变量 | 说明 |
|--------|------|
| `expr` | 表达式 |
| `ident` | 标识符 |
| `ty` | 类型 |
| `stmt` | 语句 |
| `pat` | 模式 |
| `path` | 路径 |
| `literal` | 字面量 |
| `tt` | 标记树 |
| `block` | 代码块 |

### 多模式匹配
```rust
macro_rules! my_macro {
    // 匹配单个参数
    ($arg:expr) => {
        println!("Single arg: {}", $arg);
    };
    // 匹配两个参数
    ($arg1:expr, $arg2:expr) => {
        println!("Two args: {}, {}", $arg1, $arg2);
    };
    // 匹配可变参数
    ($($arg:expr),*) => {
        $(println!("Var arg: {}", $arg);)*
    };
}
```

### 重复模式
```rust
// 使用 $(...)* 或 $(...)+ 进行重复
macro_rules! vector {
    ($($x:expr),*) => {
        {
            let mut temp_vec = Vec::new();
            $(temp_vec.push($x);)*
            temp_vec
        }
    };
}

fn main() {
    let v = vector![1, 2, 3, 4, 5];
    println!("{:?}", v);
}
```

## 过程宏

过程宏分为三种类型：

### 1. 派生宏 (Derive Macros)
用于自动实现trait。

```rust
// 在 lib.rs 中
use proc_macro::TokenStream;
use quote::quote;
use syn::{parse_macro_input, DeriveInput};

#[proc_macro_derive(HelloMacro)]
pub fn hello_macro_derive(input: TokenStream) -> TokenStream {
    let input = parse_macro_input!(input as DeriveInput);
    let name = &input.ident;
    let expanded = quote! {
        impl HelloMacro for #name {
            fn hello_macro() {
                println!("Hello, Macro! My name is {}!", stringify!(#name));
            }
        }
    };
    TokenStream::from(expanded)
}
```

使用：
```rust
#[derive(HelloMacro)]
struct Pancakes;

fn main() {
    Pancakes::hello_macro();
}
```

### 2. 属性宏 (Attribute Macros)
类似于派生宏，但可以添加参数。

```rust
#[proc_macro_attribute]
pub fn route(attr: TokenStream, item: TokenStream) -> TokenStream {
    // 处理属性宏
}
```

使用：
```rust
#[route(GET, "/")]
fn index() {}
```

### 3. 函数式宏 (Function-like Macros)
看起来像函数调用，但操作的是TokenStream。

```rust
#[proc_macro]
pub fn sql(input: TokenStream) -> TokenStream {
    // 编译时SQL解析
}
```

使用：
```rust
sql!(SELECT * FROM users WHERE id = 1);
```

## 常用内置宏

```rust
// 打印调试信息
println!("Hello, world!");
println!("Number: {}", 42);
println!("Name: {}, Age: {}", "Alice", 30);

// 格式化字符串
let s = format!("x = {}, y = {}", 10, 20);

// 断言
assert!(1 + 1 == 2);
assert_eq!(1 + 1, 2);
assert_ne!(1 + 1, 3);

// 调试断言（仅在debug模式生效）
debug_assert!(true);

// 恐慌
panic!("Something went wrong!");

// 编译时检查
compile_error!("This code should not compile!");

// 环境变量
let path = env!("PATH");

// 文件内容
const CONFIG: &str = include_str!("config.txt");

// 模块导入
mod my_module;

// 测试
#[test]
fn test_example() {
    assert_eq!(2 + 2, 4);
}
```

## 宏的卫生性

Rust宏是"卫生"的，意味着宏中定义的变量不会与调用处的变量冲突：

```rust
macro_rules! using_a {
    ($e:expr) => {
        {
            let a = 42;
            $e  // 这里的a不会与外部的a冲突
        }
    };
}

fn main() {
    let a = 1;
    let result = using_a!(a + 2);  // 这里a = 1，而不是42
    println!("Result: {}", result);  // 输出: 3
}
```

## 宏的最佳实践

1. **优先使用函数**: 如果函数可以解决问题，就不要用宏
2. **保持简单**: 宏越复杂，越难调试
3. **添加文档**: 使用 `///` 为宏添加文档注释
4. **考虑可读性**: 宏展开后的代码应该容易理解
5. **使用 `#[macro_export]`**: 如果想让宏在其他crate中使用

```rust
/// 计算两个数的最大值
/// 
/// # Examples
/// ```
/// let max = max!(1, 2);
/// assert_eq!(max, 2);
/// ```
#[macro_export]
macro_rules! max {
    ($a:expr, $b:expr) => {
        if $a > $b { $a } else { $b }
    };
}
```
