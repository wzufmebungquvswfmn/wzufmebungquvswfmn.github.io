# 字符串相关转换

# 从 `Vec::<u8>` 到 String 的转换
- `String::from_utf8(vec: [u8])`:  
    · 会夺走vec的所有权，  
    · 伴随着严格的错误处理，不允许无效u8字符，返回`Result<String, FromUtf8Error>`,  
    · 其中 FromUtf8Error 结构如下：  
    ```
    FromUtf8Error { 
        bytes, // 存储原有的数据，使其不至于丢失 
        error: Utf8Error { 
            valid_up_to: usize，  // 第一个无效字符的索引，也是最长的有效字符前缀的长度
            error_len: Option<u8> // 无效字符的长度，详见 utf8 章节
        } 
    }
    ```
- `String::from_utf8_lossy(&vec: &[u8])`:   
    · 只拿vec的借用，无条件转换，返回 `Cow<'_, str>`  
    · 若是全都有效字符，则返回 `Cow::Borrowed`，直接将原数组作为String， 无额外的拷贝与内存分配  
    · 若存在无效字符，则返回 `Cow::Owned`，将有效字符拷贝到新分配的内存中，无效字符用一个空字符 `U+FFFD` 代替  

- `String::from_utf8_unchecked(vec: [u8])`:  
    · 会夺走vec的所有权，  
    · 无条件转换，返回 `String`  
    · 无错误处理，若存在无效字符，会导致程序崩溃，但是很快  

以上[u8]均可替换为Vec<u8>或&[u8]，根据方法的参数类型而定

# 从 String 到 `&[u8]` 的转换
- `as_bytes` 方法返回字符串的字节切片 `&[u8]`，这是一个安全的操作，因为字符串在 Rust 中是 UTF-8 编码的，每个字符都由一个或多个字节组成，所以直接获取字节切片是没有问题的。
